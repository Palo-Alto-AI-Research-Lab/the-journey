# STYLE-VISUAL — визуальный язык книги на GitHub

> Синтез DR26-07-06-HUB-04 (арт-дирекция GitHub-лонгрида). Главный закон: **один приём = одно значение**. Читатель через 3-5 глав читает книгу по инерции - каждый блок всегда выглядит одинаково и значит одно и то же.

## Семафор блоков (закреплено)

| Смысл | Приём | Правило |
| --- | --- | --- |
| 📜 Цитата дня | обычный `>` blockquote под заголовком | никогда не alert |
| 🏷 Теги дня | строка `` `тег` · `тег` `` под заголовком | одна строка |
| ⏮ Ранее в The Journey | курсив, 1-3 строки | только сегодняшние нити |
| Вопрос читателю | `> [!IMPORTANT]` | ровно ОДИН на главу |
| Челлендж «попробуй сегодня» | `> [!TIP]` | 0-1 на главу |
| Цена ошибки / предупреждение | `> [!WARNING]` | только настоящий риск |
| Угадайка / мини-петля | *курсивная строка*, не бокс | бюджет боксов дороже |
| Техдеталь / логи / кухня | `<details><summary>` | второй слой, не мусорный мешок; alerts ВНУТРЬ details не класть (баги рендера) |
| Финансы / статистика / артефакты | компактная таблица | данные, не проза |
| Схема процесса | ` ```mermaid ` | когда схема живёт и меняется |
| Скрин-«квитанция» | картинка сразу после тезиса, который она доказывает | один скрин = одна мысль |

**Бюджет акцентов: 3-5 акцентных блоков на главу, из них alert-боксов - максимум 2.** (GitHub Docs сами живут по этому правилу.) Если каждые два экрана цветной бокс - глаз перестаёт понимать, что важно.

## Ритм текста (нормы)

- Абзац 2-4 предложения (40-90 слов), потолок ~120 - дальше дробить.
- Подзаголовок каждые 150-250 слов или на каждом драматическом повороте; H2/H3 в главе - 4-8. Заголовки питают Outline-меню GitHub - они и есть навигация.
- **Жирный** - дорогой ресурс: 1-3 акцента на абзац, только термины/суммы/ставки/решения (2-6 слов), никогда целые предложения.
- `---` только между макро-сценами (1-3 на главу).
- Эмодзи: один в заголовке секции - навигация; три в строке - шум.
- Таблица - для данных, никогда для вёрстки; широкие таблицы убивают мобильное чтение.

## Картинки (когда дойдём до баннеров)

- Хранить в репо `assets/chapters/<день>/`, ссылаться относительными путями.
- Каждая картинка живёт в светлой И тёмной теме: `<picture>` + `prefers-color-scheme` (или суффиксы `#gh-dark-mode-only`/`#gh-light-mode-only`).
- Alt-тексты по-русски, передают смысл в контексте; декоративное - `alt=""`.
- Разделители-картинки - нет: `---` достаточно. Один hero-баннер сверху лучше пяти «полосок».

## Жёсткие границы GitHub (не воевать с платформой)

- CSS/inline-style/class/id/цветной текст - вырезаются санитайзером. Цветного текста НЕ существует.
- Alert за alert-ом и alert внутри details - запрещено (рекомендация GitHub + известные баги).
- `#123` в файлах НЕ становится ссылкой на issue - только явные ссылки.
- Автоякоря заголовков ломаются при переименовании - мини-оглавления проверять после правок заголовков.
- Проверять каждую главу в светлой и тёмной теме - это не edge case, а базовый сценарий.

## Top-5 делай / никогда

**Делай:** семантика за каждым приёмом · пиши для сканирования · details = второй слой · theme-aware картинки · assets в репо.
**Никогда:** стиль на CSS · alert за alert-ом · таблица как роман · alerts внутри details · картинка без проверки в двух темах.

## ⏫ UPD (2026-07-06): дельты из второго отчёта DR26-07-06-HUB-04

Второй вендорский отчёт независимо подтвердил семафор, бюджет боксов и ритм-нормы (цифры совпали). Новое, что принимаем:

- **Мини-оглавление «В этой главе»** - опция для глав длиннее ~1500 слов: список из 4-6 ссылок на секции сразу после холодного опена. Короткие главы - без него.
- **Якоря словаря заморожены**: все главы ссылаются на заголовки `GLOSSARY.md` - после публикации заголовки статей словаря НЕ переименовываем (переименование = сломанные ссылки во всех главах). Новое имя = новая статья + старая остаётся алиасом.
- **Скрин-квитанция, полный протокол**: перед скрином строка «что подтверждает этот скрин», после - вывод одной строкой; резать браузерный хром и соседние виджеты.
- **Ссылка на строку файла**: `?plain=1#L14` - постоянная ссылка на конкретную строку markdown-артефакта (для «артефакт → место в главе»).
- **Спор разрешён - остаёмся при своём**: отчёт предлагал «Вопрос читателю» заголовком `### ❓`, не боксом. Решение: вопрос остаётся `> [!IMPORTANT]` (уже раскатан на все главы; это наш единственный «дорогой» акцент, IMPORTANT рендерится «важно», не «опасно»).
- Не берём: MathJax-формулы (нет формул - нет «умного вида» ради вида).

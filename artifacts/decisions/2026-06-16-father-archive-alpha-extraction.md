# Решение: как вынимать ценное / альфу из архива писем отца

> [!abstract] Alpha Protocol, шаг 4 — SYNTHESIS
> Recall (мой Второй Мозг) + Deep Research (внешний отчёт Антона, `parent deep-research-report.md`) → этот Decision Memo. Решаем НЕ «строить чат-бот-отца», а превратить архив в **квот-привязанный граф утверждений** — невыдуманный, проверяемый слой знания, который потом кормит цифрового двойника. См. [[insight-glavnaya-tsel-tsifrovoj-dvojnik|Главные цели]] #1, [[_Self-Bible-MOC|Self-Bible identity-слой]].

## 1. Проблема

Архив переписки с отцом собран и разложен ([[_Perepiska-s-Ottsom-MOC]] — 623 письма, 2015–2020; саммари [[Mudrost-ottsa-summary]]). Вопрос: **как из него системно вынимать ценность** — мировоззрение, принципы, советы, фактуру рода — так, чтобы это было (а) невыдуманным и проверяемым, (б) кирпичом цифрового двойника, а не разовой прозой.

## 2. Что мы уже знаем (Recall)

- **Слой-доказательство уже есть:** годовые заметки + `dump\messages.json` (812 копий → 633 уникальных по RFC `Message-ID`), у каждого письма дата, отправитель, текст, вложения. Провенанс: письма отца `origin: отец`, ответы Антона `#anton-original`.
- **Первая интерпретация уже есть:** [[Mudrost-ottsa-summary]] — проза (мировоззрение, 12 тем, принципы, 15 цитат). Это v0 — НЕ привязано к точным цитатам построчно.
- **RAG уже есть:** `brain_ask.py` (e5 + reranker) — архив переиндексирован, отвечает «архивным режимом» уже сейчас.
- **Движок «дешёвый детектор → дорогой судья» уже есть:** [[decision-alpha-extraction-engine-variant-a|Alpha-extraction engine]] (10 майнеров) + [[insight-prediction-ledger]] — готовый дом для решений/прогнозов Антона.
- **Север:** [[insight-glavnaya-tsel-tsifrovoj-dvojnik|Главная цель #1]] #1 = цифровой двойник; влияние отца на Антона = identity-данные двойника.

## 3. Что сказал Deep Research (ключевое)

1. **НЕ «обучать бота-отца». Ядро = граф «утверждение ↔ доказательство»**, а не беглый симулякр. Каждое верование/совет/черта → точная цитата + дата + контекст треда + уверенность.
2. **Три слоя:** (A) Доказательство (сырьё+спаны), (B) Интерпретация (ценности/принципы/противоречия/связи с провенансом), (C) опциональный интерфейс-наследие (отвечает ТОЛЬКО из слоя A, показывает неуверенность, не выдумывает от первого лица).
3. **RAG-first, НЕ fine-tuning** для корпуса такого размера — ретрив сильнее дообучения на малых личных данных и безопаснее (дообучение «впечатывает» галлюцинации и делает их убедительнее).
4. **Два РАЗДЕЛЬНЫХ архива + мост:** Архив-А «Отец-как-доказательство», Архив-B «Ты-во-времени» (твои ответы → ledger решений/прогнозов/эмоций), Мост = «влияние» (его совет → твой последующий шаг).
5. **Твои ответы — это не «эмоции», а ledger решений/прогнозов/намерений** (что решил, что предсказал, что вышло) → бэктестируемая автобиография для двойника.
6. **Этика для архива умершего:** по умолчанию режим «архив/интерпретатор» (3-е лицо), а НЕ «реинкарнация от 1-го лица»; раскрытие, что это ИИ; стюардство; путь к «уходу» системы. Риск: люди ценят «как живой» выше точности → давление в сторону выдумки.
7. **Главные провалы:** галлюцинированная уверенность в горе; разрыв личности при смене модели (version-pin!); стилистический «отмыв» (звучит как он, но говорит чего он не говорил); ложное доверие к красиво оформленным «цитатам» без верификации; утечка приватности третьих лиц из писем.

## 4. Синтез — DR подтверждает нашу архитектуру

DR почти один-в-один ложится на то, что у нас уже построено: его 3 слоя = наш [[concept-second-brain-ingest-pipeline|3-слойный Второй Мозг]] (SQLite-факты + RAG-смысл + markdown-мысли). Слой A = ✅ готов. RAG-first = ✅ уже есть. «Дешёвый детектор → судья» = ✅ уже есть движок. **Разрыв ровно один:** слой B сейчас — проза (Mudrost), а нужен **структурный квот-привязанный граф утверждений** + **ledger решений из ответов Антона** + **мост влияния**. Persona-интерфейс (talk-to-dad) пока НЕ нужен и по этике откладывается.

**Проверка на реальных данных (детектор 1-й стадии уже работает):** грубый список cue-слов вытащил из 415 писем отца настоящие принципы, напр. (2020-01-26): «всегда помни и не забывай никогда "четыре принципа"… где? когда? с кем? и — сколько!». Полный cue-набор + LLM-судья на каждого кандидата = строки графа.

## 5. ДО → ПОСЛЕ (на реальном письме отца)

**ЧТО:** превратить прозу-саммари в проверяемую таблицу «принцип ↔ цитата ↔ дата ↔ уверенность».

- **ДО (Mudrost, проза):** «Отец ценил рассудительность и системность в решениях.» — красиво, но не проверишь, не отфильтруешь, не посчитаешь повторы.
- **ПОСЛЕ (строка ledger-а):**
  | принцип | цитата (дословно) | дата | сила | повторы | тон |
  |---|---|---|---|---|---|
  | Перед действием проверь контекст: где/когда/с кем/сколько | «всегда нужно помнить в своей жизни, в любой ситуации: где? когда? с кем? и — сколько!» | 2020-01-26 | директива | 1 (+пара эхо) | наставление |

  → теперь это можно ранжировать (повторы×абстракция×переносимость×директивность×«ты потом это применил»), искать, аудировать на невыдуманность, и связать с твоим последующим решением.
- **Что сломается / риск:** ничего из существующего (Mudrost остаётся как «человеческое» предисловие); добавляется один новый узел-таблица. Цена — один прогон детектор→судья (грунт на Sonnet, финальная вычитка Opus).

## 6. Опции

- **Опция 1 — оставить как есть** (проза Mudrost + RAG). Дёшево, но не проверяемо, не кормит двойника структурно. ❌ не тянет на цель №1.
- **Опция 2 (РЕКОМЕНДАЦИЯ) — построить слой B поэтапно по DR**, RAG-first, без бота. ✅ AK-47: каждый кирпич — простой markdown-ledger (+ опц. крошечный `father.db`), который Антон откроет глазами.
- **Опция 3 — сразу persona-бот «поговорить с отцом».** ❌ по DR — самый рискованный (галлюцинации в горе, выдумка, version-drift); откладываем до готовых слоёв A+B и явного решения по этике.

## 7. Рекомендация — 3 фазы (ценность×скорость)

1. **✅ Фаза 1 СДЕЛАНА (2026-06-16): [[Otets-Printsipy-Ledger|Ledger принципов отца]]** — 225 квот-привязанных принципов, **каждая цитата проверена дословно в письмах (225/225, 0 выдуманных)**; 14 главных принципов в ядре + полная таблица по темам. Метод: детектор (0 токенов) → 6 Sonnet-агентов (грунт) → верификация цитат → синтез на Opus. Точно по DR (квот-граф, RAG-first, без бота).
2. **✅ Фаза 2 СДЕЛАНА (2026-06-18): [[Anton-Reshenia-Ledger|Ledger решений/мыслей Антона]]** — 79 событий из 188 ответов (решение/прогноз/намерение/эмоция/просьба/рефлексия), **79/79 цитат проверены дословно**; 5 бэктест-прогнозов + 27 событий уже со связью к совету отца (`re_father`). Метод тот же: детектор → 3 Sonnet-агента → верификация → синтез Opus.
3. **✅ Фаза 3 СДЕЛАНА (2026-06-18): [[Otets-Vliyanie-Most|Мост влияния отец → Антон]]** — 27 пар «совет отца → отклик Антона», вердикт по каждой (принял 8 · переосмыслил 4 · сопротивлялся 8 · просил 3 · эмоция 4). Вывод: структурные советы (имя, дети, бизнес-форма, дом, найм) принимал; советы про темп/оседлость/близость/скромность — отвергал с формулой «у меня есть цель». Это и есть identity-функция отклика для двойника.
4. **Отложено по этике/DR:** persona-режим от 1-го лица; fine-tuning (не нужен — RAG-first). Version-pin модели когда дойдём до интерфейса.

# Decision Memo: лейн «Намерения» + Хабр

> Alpha Protocol synthesis: recall ([[content-factory]] v2 §7.2) + Deep Research (`_originals\2026-07-02-deep-research-intention-content-and-habr.md`). Статус: **proposed**, ждёт «+» Антона на стройку.

## 1. Проблема / зачем
Ежедневные НАМЕРЕНИЯ Антона (что сегодня пытаюсь сделать / что понял / кто нужен) превратить в отдельный тип контента с явной ПРОСЬБОЙ, чтобы аудитория возвращала АЛЬФУ: решение, тёплое интро, предупреждение, соавтора. Плюс отдельный вопрос: как «много постить на Хабре».

## 2. Ключевой вывод DR (снимает противоречие)
Намерения-с-просьбой и Хабр это **РАЗНЫЕ лейны**, их нельзя смешивать:
- **Пост-намерение (ask) = НЕ для Хабра.** У Хабра нет формата «спросить совета»; прямая просьба в ленте не отзовётся. Хабр = площадка для СТАТЕЙ-кейсов, не для мгновенных ответов.
- **Хабр = канал дистрибуции лонгрида/дев-лога** (технический deep-dive), уже заложен как фаза-2 в [[content-factory-v2-tiers]]. «Постить много» решается либо корпоративным блогом (платно, без инвайтов), либо личным грайндом кармы.

## 3. Лейн «Намерения» (RU + EN)
**Формат поста-намерения** (по DR, уверенность established): контекст (что делаю) → что не вышло → **явный конкретный ask** в конце. Тон честный, уязвимый, без рекламного лоска. Конкретная просьба («выбираю A или B», «нужен эксперт по X») бьёт размытую («подскажите что делать»).

**Приватность (established):** без точных сумм/выручки/данных клиентов и несогласованных имён. Обобщать («$X», «крупный лид»). Совпадает с draft-first правилом §7.3.

**Каналы намерений по «альфе» (не по охвату), ранжир DR:**

| Канал | Тип альфы | Примечание |
|---|---|---|
| **X** (EN, thread, #buildinpublic) | быстрый совет + интро, большой охват | конкуренция за внимание высокая |
| **Telegram** (RU: ClawRus + профильные IT/стартап-чаты) | прагматичные ответы | аудитория меньше, но тёплая |
| **Indie Hackers** (EN) | обстоятельные советы, контакты | не рекламировать продукт |
| **Hacker News** (EN, «Ask HN:») | высокое качество советов | трудно пролезть на главную |
| **LinkedIn** (EN, деловой тон) | рекомендации экспертов/интро | контекстуализировать |
| **Facebook** (только нишевые группы) | низко-средне | вне групп не работает |
| ⛔ **Хабр / VC.ru** | НЕ для ask | Хабр = статьи; VC.ru не заточен под Q&A |

**Старт (АК-47):** X + Telegram (уже живые каналы). Расширение: Indie Hackers + «Ask HN» + LinkedIn по мере готовности.

**Частота:** 1 намерение/день допустимо, если РАЗНООБРАЗИТЬ формат (вопрос / победа / ретро / опрос), иначе аудитория устаёт (DR: speculative). Смотреть на реакции.

**Петля альфы (замыкание):** откликнувшиеся → благодарность → перевод в личку → в волт + CRM (`leads.db`) как новые интро/лиды. Это оживляет давно висящий alpha-capture лейн ([[content-factory]]).

**5 шаблонов** (RU+EN, готовы в DR §A4): нужен эксперт по X · выбор A/B · уперся в стену · понял, что ошибался · общая помощь/опрос. Кладём в style-файл адаптера.

## 4. Хабр как канал (отдельно от намерений)
- Новый аккаунт: читать/комментировать/писать в **Песочницу**; без инвайта в основную ленту не попасть. Инвайт при **50 кармы** / статья с рейтингом >50. Песочница модерируется (проверка AI-контента, плагиата).
- **Личный vs корпоративный блог:** корпоративный = платно (мин. 3 мес, от ~121 тыс. руб), несколько авторов, БЕЗ инвайтов. Личный = бесплатно, но грайнд кармы.
- **Что заходит (плюсуют):** технические кейсы, туториалы с кодом, разборы/оптимизации, постмортемы «как мы сделали X», числовые результаты, TL;DR, заголовки-вопросы, до 5 релевантных хабов. **Минусуют/банят:** прямую/замаскированную рекламу, тонкий гуглящийся контент.
- **Козырь Антона:** PhD + reality-show технические лонгриды = ровно то, что Хабр плюсует. Хабр-контент = переупаковка longread/dev-log, канон → GitHub.

## 5. РЕШЕНО (Антон, 2026-07-02)
1. **Хабр-маршрут:** **личный аккаунт бесплатно** сейчас, первая статья-кейс из reality-show → грайнд кармы до инвайта. Корпоративный блог (~121k руб) оценить позже по данным.
2. **Каналы намерений на старте:** **X + Telegram + Indie Hackers + Ask HN** (RU на TG, EN на X/IH/HN).
3. **Захват намерения = АВТО-МАЙНИНГ ИЗ ВСЕХ СЕССИЙ ДНЯ** (НЕ утренний вопрос). Механика (правило Антона):
   - Каждая боль/вопрос юзера в сессиях («как сделать X», «нужен человек, кто…», «не знаю как») = намерение.
   - Майнер читает ВСЕ сессии дня (как `fb_diary_collect`, через `vault_sessions`), извлекает 5-10 намерений/день.
   - Складывает в **КОПИЛКУ НАМЕРЕНИЙ** (`intentions.db`, никогда не теряет, как task_registry).
   - Постим **2-3+ намерения/день** (можно и все) по разным каналам, draft-first.
   - **Описывать МАКСИМАЛЬНО ПОДРОБНО:** как юзер к этому пришёл, зачем, контекст боли (+ явный ask). Приватность §3 в силе (без сумм/имён).

## 6. Рекомендация / что делаем
- Ввести **пост-намерение** как мини-формат тира teaser/medium: контекст→что не вышло→ask, RU(TG)+EN(X), draft-first, голос=Opus. Каналы: X+TG на старте.
- **Не** класть намерения в Хабр/VC.ru/в тело FB.
- Хабр вести отдельно как longread/dev-log канал: старт личным аккаунтом, первая статья-кейс.
- Замкнуть петлю: отклики → CRM.

## 7. Риски / уверенность
- Формат ask-постов: **high** (established).
- Ранжир каналов: **medium** (emerging, зависит от конкретных сообществ).
- Частота 1/день: **low-medium** (speculative) → проверять на реакциях.
- Хабр-механика (карма/инвайт/цена корпблога): **medium**, `volatile` → сверить перед действием.

# 🅰️ Decision Memo — «Движок альфы» для Второго Мозга

> Итог Alpha Protocol (Recall + Deep Research + Synthesis) по теме «как доставать альфу из собранного контента». Протокол: [[protocol-alpha-protocol-recall-plus-deep-research]]. Это кирпич Цели №1 — цифрового двойника ([[concept-digital-immortality]], [[insight-glavnaya-tsel-tsifrovoj-dvojnik]]).

## Проблема
Второй Мозг (~154k заметок) силён в **хранении и поиске**, но не умеет **сам замечать** ценное: противоречия, смену мнений, забытые ставки, кросс-доменные связи. Цель №1 требует системы, которая выдаёт инсайты **без вопроса**.

## Текущие знания (RECALL)
3-слойка уже есть и DR велит её сохранить: SQLite-факты + RAG e5+reranker (`brain_ask.py`) + markdown-волт + дистилляция (concepts + reference-cards) + identity-слой + провенанс во frontmatter + токен-закон. Слой времени (`epistemic-decay`: valid_as_of/volatility) — пока идея, DR говорит реализовать.

## Результаты DR (главное)
- **Центральный вывод (= наш gap):** «перестань оптимизировать „как лучше спросить“ — оптимизируй „что система замечает сама“».
- **Архитектура:** не ломать стек, добавить **4-й слой «insight engine»** = temporal claim/decision ledger (SQLite) + лёгкий доменный граф + ночной детерминированный майнер-кандидатов + LLM-писатель только на топ-кандидатах (= DR-паттерн «дешёвый детектор → дорогой писатель» = наш AK-47/токен-закон).
- **Инструменты под наше железо (RTX A3000):** SQLite FTS5 + DuckDB + igraph/Leiden + Ollama/llama.cpp, модель **7B+** (3.8B не тянет структурный вывод).
- **Провенанс — first-class** (у нас уже есть). **Не строить один гигантский граф** — тонкие доменные. **Не подмешивать веб-контекст** в извлечение мнений (искажает). **Не оптимизировать под бенчмарки QA.**

## Найденная инфраструктура (read-before-build, 2026-06-14)
`E:\Obsidian\_imports\alpha\alpha_scan.py` уже существует — детерминированный пре-фильтр **только по AI-чатам** (Claude-AI/ChatGPT), типы: идея/решение+почему/фреймворк/**прогноз**/**открытая петля**. Это родственник (тот же паттерн), но **уже** Варианта A (нет ledger'а/полярности/дат-ставок, охват только чаты, нет еженедельного ≤10-отчёта с да/нет). **Решение: переиспользовать его паттерн и типы, не дублировать и не править сам файл (collision-риск с возможной параллельной сессией).**

## Каталог ВСЕХ майнеров (полный)
Ядро (из DR, 6):
1. **Contradiction** — противоречия (same-subject/opposite-polarity, оба «текущие»).
2. **Stance-shift** — смена позиции/мнения во времени (change-point по топику).
3. **Forgotten-bets** — забытые ставки/прогнозы/планы с датой, не пересмотренные. ⬅️ **выбран первым**
4. **Cross-domain bridge** — кросс-доменные мосты (bridge-узлы, высокий betweenness).
5. **Recurring-pattern** — повторяющиеся мотивы/паттерны через годы.
6. **Lead-signal** — лид-сигналы (CRM-join: похож на прошлые выигрыши/проигрыши).

Расширения (наши, под двойника — не из DR, опционально позже):
7. **Novelty/anomaly** — что реально ново vs историческая база (не банальность).
8. **Identity-consolidation** — устойчивые убеждения → в identity-слой двойника (feeds Цель №1).
9. **Open-question** — открытые вопросы, которые Антон сам себе задал и не закрыл.
10. **Orphan-insight** — ценные мысли без связей (стык с [[relink-mechanism]]/[[no-orphan-notes-rule]]).

## Варианты
- **A — один майнер, ручка-молоток (ВЫБРАН).** ledger (claim+decision) + ОДИН майнер + еженедельный ≤10-отчёт с кнопкой да/нет. Дёшево, проверяет «попадаем ли в ценное».
- B — полный insight-engine (все 6 + граф + InsightScore). ⚠️ месяцы, поддержка графа.
- C — взять готовое (Cognee/Zep-Graphiti/Khoj). ⚠️ все под QA/resurfacing, «последнюю милю» всё равно самим; многие cloud-first.

## Риски (DR failure modes)
1. «Отвечает, но не замечает» (лечат майнеры). 2. «Схлопывание времени» (нужен valid_from/valid_to). 3. Перестройка графа (тонкие доменные). 4. Веб-контекст как скрытая предвзятость (свой корпус первым). 5. Тривиальные инсайты (importance+novelty вместе).

## РЕШЕНИЕ
**Вариант A. Первый майнер — Forgotten-bets** (самый чистый сигнал «попали/нет», очевидная ценность). Строим как сиблинг, переиспользуя паттерн `alpha_scan.py`. Реализация — только после этого Memo (Alpha Protocol соблюдён). Ночной прогон — кандидат на рутину ([[evaluate-recurring-into-routine]]).

## Статус реализации (2026-06-14)
- **Forgotten-bets майнер v2 построен** = `_imports\alpha\bets_scan.py` (сиблинг alpha_scan.py, не трогает его). Сканит Concepts/Insights/Claude/Claude-AI/ChatGPT.
- **Конвейер доказан end-to-end:** детерминированный детектор (2397 заметок → 74 кандидата → 32 дозревших, 0 токенов) + **LLM-судья** (32 → 4 реальных ставки Антона). Это DR-паттерн «дешёвый детектор → дорогой судья».
- **Вывод:** чистый regex упирается в потолок precision (~3 итерации v0→v2: режет телеметрию/таймлайны/ledger-мету, но не отличает «мою ставку» от «факта в research-заметке»). Precision добивает дешёвый LLM-судья на ~десятках кандидатов — НЕ дальнейшая прокрутка regex.
- **Решение по дому:** майнер НЕ строит свой ledger — **кормит существующий [[insight-prediction-ledger]]** (+ движок `ledger_calibration.py`). Найденные ставки (#17 CHARM TGE Q4'25, #20 стейблкоин-2020, #19 qdefirating-2021) = кандидаты на добавление.
- **Открыто:** автоматизировать судью (локальная 7B+ модель Ollama/llama.cpp на RTX A3000, per токен-закон) для недельной рутины; календарь maturity-nudge по ⏳-строкам реестра.

## Дорожная карта всех 10 майнеров (одобрено Антоном 2026-06-14)
Строим ПО ОДНОМУ как клоны одного скелета (детектор → дом → недельный ≤10 → кнопка да/нет → калибровка). #3 первый (в работе). Порядок далее — по ценности×простоте, уточняется. Каждый «кормит» свой дом, не плодит дубль:
1 Contradiction · 2 Stance-shift · **3 Forgotten-bets (в работе → insight-prediction-ledger)** · 4 Cross-domain bridge · 5 Recurring-pattern · 6 Lead-signal (→ Platinum CRM) · 7 Novelty/anomaly · 8 Identity-consolidation (→ identity-слой двойника) · 9 Open-question · 10 Orphan-insight (→ /relink).

## Addendum (2026-06-16, сессия «chat-corpus») — второй независимый DR ПОДТВЕРДИЛ
> Добавлено из параллельной сессии (фокус: альфа именно из ChatGPT+Claude.ai чатов/артефактов). Авторство аддендума: claude. Источник DR: [[2026-06-16-alpha-deep-research-report]] (оригинал в `_originals\alpha-deep-research\`). Это **второй, независимый** Deep Research — и он сошёлся с первым (ниже), что усиливает уверенность.

**Сходимость двух DR (главное):** оба вывели одно — «classify+score = лёгкая половина; настоящая альфа = conflict / bridge / temporal-drift / gap / hypothesis; детерминированный детектор-кандидатов → LLM только на шорт-лист; провенанс first-class; eval с 1-го дня». Это валидирует Вариант A/дорожную карту 10 майнеров. (Моя рамка «Вариант C — staged hybrid» = ровно этот Вариант A: дешёвые детекторы сейчас, claim-warehouse+граф ⚠️ позже.)

**Что 2-й DR добавляет к реализации (берём бесплатно, во ВСЕ майнеры):**
- **Академическая основа:** literature-based discovery (Swanson ABC) · GraphRAG→LazyGraphRAG (−77% стоимости, «не строить большой граф») · Claimify (атомарные claims) · Uzzi/Shi-Evans (новизна = «обычный костяк + одно нетипичное ребро», НЕ экзотика) · VeriTrail (трассируемость на каждом шаге) · «Lost in the Middle» (не «скорми весь волт»).
- **Консервативный рубрик скоринга кандидата:** `novelty` (рекомбинантная дистанция, не семантическая экзотика) · `evidence-diversity` (разные типы источников) · `actionability` (меняет решение/архитектуру заметок) · `epistemic-risk` (штраф за слабый провенанс / высокую долю LLM). Правило: **interestingness НИКОГДА не бьёт provenance.**
- **Eval-харнес (ядро-метрика поведенческая):** precision@10/20 по майнеру · accepted-into-permanent rate · edit-distance(всплывшее→сохранённое) · revisit@30d · паттерны false-positive. Главный вопрос: **всплытие реально привело к обновлению знания?**
- **Дисциплина «скептик по умолчанию»** (critique → re-rank → validate → trace) — как у Co-Scientist/Claimify.

**Находка по инфраструктуре (сессия chat-corpus, 2026-06-14):** `alpha_scan.py` доведён до 2-тирового детермин. фильтра (501→370; пойман баг подстроки «тент»←«кон**тент**», «мульт»←«**мульт**иаккаунт» → починен word-boundary). Эмпирически подтверждён вывод обоих DR: **ключевые слова альфу не СУДЯТ (это смысл), они лишь убивают явный шум — воронку делает дешёвый LLM-скоринг.** `alpha_scan.py` = AI-chat пре-фильтр (сиблинг `bets_scan.py`, файлы друг друга не трогают).

**Открытый пункт реконсиляции (на решение Антона):** что строить следующим, чтобы НЕ дублировать майнеры параллельной сессии — (a) оформить `alpha_scan` как chat-майнер в общем скелете, (b) следующий майнер дорожной карты (#1 Contradiction / #9 Open-question), или (c) **единая review-поверхность + eval-харнес** (дашборд + `/alpha` + кнопка accept/reject), куда сливаются ВСЕ майнеры (рекомендация 2-го DR «surfacing + eval с 1-го дня»; не дублирует ни один майнер).

# Decision Memo — публичная продуктизация системы

> ⚡ **SUPERSEDED тем же днём (2026-07-02, 12:22):** Антон разобрал все 14 возражений голосом и перевернул вердикт, сменив рамку с «коммерческий продукт/SaaS» на «бесплатно + обучение + контент для людей и роботов, self-hosted». Действующее решение → [[decision-open-second-brain-free-education-go-2026-07-02]]. Этот мемо сохраняется как история: его DR-вход и границы («что не публикуем») перекочевали в GO-решение как инварианты.

> Альфа-протокол, Шаг 4 (Synthesis). Recall + 1 раунд DR ([[dr-public-productization-2026-07-02]]). `origin: anton` — решение Антона, DR = вход. Отдельно от уже принятого решения по friends-стартер-киту ([[decision-friends-starter-kit-fork-and-concierge]]) — то не трогаем, оно продолжается как есть.

## Проблема
Если "второй мозг" (Claude Code + Obsidian + CRM + live multi-channel actor + multi-machine governance + persona/twin-слой) упаковать как скилл/тулзу для продуктивности — стоит ли выпускать это ПУБЛИЧНО, для чужих людей (не для друзей), и если да — куда класть.

## Текущие знания (recall)
- PKM/second-brain рынок уже насыщен бесплатными зрелыми китами (claude-obsidian ~6-8k⭐, obsidian-second-brain ~2.9k⭐, kepano/obsidian-skills ~38.7k⭐) — см. [[dr-friends-starter-kit-2026-07-02]].
- Сегодня же принято отдельное решение для УЗКОГО кейса (друзья): форк claude-obsidian + приватный репо + concierge — [[decision-friends-starter-kit-fork-and-concierge]]. Публичный маркетплейс там explicitly отклонён как преждевременный.
- [[decision-persona-platform-roadmap]] (2026-06-14) заранее пометил: архитектура персоны намеренно AK-47-простая для ОДНОГО пользователя (Антон+я); `revisit_if`: "появится второй пользователь продукта — тогда вернуться к варианту B (полная persona-platform)".
- Цель №1 (`main-goals`) — цифровой двойник ДЛЯ АНТОНА, не SaaS-бизнес по умолчанию.

## Результаты DR — см. [[dr-public-productization-2026-07-02]]
PKM-часть рынка насыщена и не наша игра. Уникальное — не PKM, а бандл (live-actor + CRM + governance + persona), которого ни у кого нет — но именно он несёт максимальный риск (агент шлёт реальные сообщения от чужого имени = доверие+юр.ответственность), а архитектура НЕ рассчитана на мультитенант. Retention для похожих solo-builder digital-twin продуктов низкий. Дистрибуция через Claude-маркетплейсы не подходит — система не влезает в формат простого skill.

## Варианты
- **A. Широкий публичный consumer-запуск сейчас.** ОТКЛОНЁН DR: насыщенный рынок + архитектурный mismatch (single-tenant) + высокий support/trust/legal груз + мы не готовы взять на себя роль "агент шлёт сообщения от лица незнакомца".
- **B (ПРИНЯТ, по умолчанию — статус-кво).** Ничего сверх уже идущего: система остаётся приватной, single-user (Цель №1), + уже запущенный friends-концьерж ([[decision-friends-starter-kit-fork-and-concierge]]) ПРОДОЛЖАЕТСЯ как есть — DR сам называет "beta-тест на 5-10 доверенных юзерах" правильным первым шагом, а это уже друзья-кит, отдельного нового действия не требуется.
- **C. Нишевый B2B/enterprise (AI sales-assistant с CRM+messaging).** ОТЛОЖЕН, не начат. Требует: мультитенант-передел, security/legal review, hosted SaaS упаковку, open-core (движок открыт, backend-оркестратор+persona-кодекс закрыты) — это отдельная, крупная инвестиция, не мелкая доработка. Не стартуем без явного сигнала (см. `revisit_if`).

## РЕШЕНИЕ (B, статус-кво + C явно отложен)
1. **Публичного продукта для чужих людей сейчас не строим.** Никакого нового движения не требуется — friends-концьерж уже покрывает рекомендованный DR "beta с доверенными юзерами".
2. **Что НЕ открываем НИКОГДА, даже если когда-то пойдём в C:** коннекторы Gmail/Telegram/WhatsApp (бизнес-логика), протокол multi-machine governance, persona-кодекс/Библия. Это защитный актив (moat), уже и так закрыт по умолчанию — просто фиксируем правило на будущее.
3. **`revisit_if` для варианта C** — см. frontmatter выше. Пока ни одно условие не выполнено.
4. **[[decision-persona-platform-roadmap]] не пересматриваем** — его `revisit_if` ("второй пользователь продукта") НЕ сработал, публичного второго пользователя не будет.

## Риски и лечение
- Соблазн "быстро выложить в маркетплейс" ради видимости прогресса → лечится этим решением: явный NO-GO, не тратим время, пока нет реального сигнала.
- Конкуренты (Gemini/Copilot/сам Anthropic Cowork) могут закрыть нишу раньше → не наша битва сейчас (см. Цель №1 — это инструмент ДЛЯ Антона, не гонка за рынком).

## Что дальше (не начато)
Ничего нового не запускаем. Продолжаем: (a) friends-концьерж по уже принятому плану; (b) Цель №1 (цифровой двойник для Антона). Если появится конкретный B2B-сигнал — тогда новый раунд Alpha Protocol именно под этот сигнал (не абстрактный "а не продать ли").

## Ключевой поворот рамки (почему NO-GO не применим)
DR-A отвечал на вопрос «строить ли КОММЕРЧЕСКИЙ продукт/SaaS для чужих». Антон делает ДРУГОЕ:
- **Бесплатно, non-profit.** Не продукт, не подписка, не hosted SaaS. Даём людям систему + обучение даром.
- **Цель = репутация и посев знаний**, не revenue. Кейсы + экспертность → чтобы заметили (Anthropic/OpenAI/рынок), звали, брали на работу, покупали экспертизу. «Мне нужны кейсы, а не retention».
- **Контент для людей И для роботов.** Люди читают посты Антона и находят альфу; агенты («эй, Claude, посмотри что Антон пишет, найди альфу») майнят наш открытый бэклог. Уже есть раздел «контент для роботов» на GitHub. Каждая публикация = приманка для обоих.
- **Self-hosted, не мультитенант.** Мы не хостим ничьи данные. Каждый юзер разворачивает СВОЮ single-tenant копию на своей машине со своими аккаунтами: «мы просто приклеили соцсетки к Claude Code, юзер сам авторизует». Ответственность за отправку сообщений = у юзера, как у любого OSS-коннектора (telegram-mcp и т.п. уже годы публичны).
- **Саппорт = Claude Code юзера + наша документация.** «Лучше Claude Code я всё равно не отсапорчу». Мы пишем супер-доку (уже пишем), юзер кастомизирует своим Клодом сам.
- **Формат = нарезка на плагины-семью.** Систему режем на кучу самодостаточных плагинов; каждый говорит «я из семьи Антона и Майка, дружу с такими-то скиллами» (правило Connect между скиллами); Библия = карта всего. У других вата, у нас скелет.
- **Killer-кейс = sales-copilot.** Сообщение Пете пишется со знанием всей истории переписки + доп-ресёрч = суперперсонализация, которую юзер сам не напишет. Общение с лидами = продажи = деньги (юзера). Это наша уникальная связка CRM+outreach.

| 8 | Приватность под лупой | ✅ Растворено для юзеров (их данные у них). ⚠️ ОСТАЁТСЯ для НАС: наша система пропитана данными Антона → hard-gate leak-scan (см. инварианты). |
| 9 | Retention 30-40% | ✅ Растворено: «похер, мне нужны кейсы». Метрика успеха = кейсы/репутация/входящие, не MAU. |
| 10 | Саппорт разорвёт | ✅ Растворено: саппортит юзеров их собственный Claude Code + наша документация. Мы не обещаем поддержку. |

| 12 | Крупные игроки закроют нишу | ✅ Растворено: «пусть едут». Для миллионов ≠ для думающих продажников-специалистов. А если крупный игрок нас заметит и купит/наймёт = это и есть цель. |
| 13 | Смена цели №1 | ✅ Растворено: не смена, а расширение. Технология, работающая только на одном юзере = плохая технология; обкатка на «разных мне» (+двойники напарниц) закаляет двойника. Синтетическая личность Майка = уже мульти-генная. |
| 14 | Нет входящего спроса | ✅ Растворено: посев ЕСТЬ стратегия создания спроса (контент-фабрика уже работает ежедневно). |

## Что ОСТАЁТСЯ жёсткими инвариантами (не растворено, это границы, не блокеры)
1. **Leak-scan перед каждой публикацией** (⛔ hard gate): наши репо/Библия/скиллы пропитаны личными данными (CRM-содержимое, имена, суммы, секреты, session-логи). Публикуем каркас/движок/уроки, НИКОГДА данные. Канонический инструмент: `E:\Obsidian\_imports\leak_scan.py` (нормализует пробелы/тире перед матчингом - закрывает класс «форматирование обходит сканер»; exit 1 = BLOCKED; собран и протестирован подсадными утками 2026-07-02) + ручной просмотр.
2. **Секреты вне репо** ([[credential-store]] неизменен).
3. **Публикация = Tier-2 наружу**: ~~каждый новый публичный репо/плагин перед первым пушем показывается Антону~~ ⭐ UPDATE (anton, 2026-07-03, после чистых пушей №1 и №2): «отныне ПОСТИ все сам на гитхабе — полностью ДОВЕРЯЮ» → GitHub-публикации движения автономны, без показа перед пушем. Leak-scan (инвариант 1) остаётся ⛔ hard gate перед КАЖДЫМ пушем; мандат только про GitHub — Telegram/FB/каналы сохраняют свои ворота.
4. **Время Антона+моё = единственная реальная цена.** Не бесконечная нарезка ради нарезки: режем и публикуем порциями по ценности (AK-47), каждая порция сопровождается контентом (пост = кейс).

## РЕШЕНИЕ (GO в рамке «бесплатно + обучение + контент для людей и роботов»)
1. Систему нарезаем на самодостаточные публичные плагины/скиллы («семья Антона и Майка»: каждый декларирует родство и связи-Connect с другими).
2. Библия-фреймворк (обезличенный скелет, не содержимое) = карта семьи плагинов и главный дифференциатор («у других вата, у нас скелет»).
3. Дистрибуция по DR-B: GitHub-репо с валидной схемой → CrossAI авто-индекс (даром) → awesome-list при 10⭐ → официальный маркетплейс Anthropic как амбиция.
4. Каждый релиз = контент-волна (пост RU/EN через контент-фабрику + раздел «для роботов» в репо).
5. Friends-концьерж ([[decision-friends-starter-kit-fork-and-concierge]]) продолжается и становится бета-полигоном открытой версии.
6. NO-GO-мемо того же дня помечен superseded, но его DR-вход сохраняет ценность (границы «что не публикуем» пришли оттуда).

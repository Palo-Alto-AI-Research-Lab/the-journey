# Decision Memo — сайдбар сессий приходит из облака

> Триггер: три захода «почини список чатов» чинили не тот слой — перезапуск ничего не менял, время горело. Anton подтолкнул к ресёрчу.

## Находка
Два внешних Deep Research сошлись (~90%): список сайдбара «Recents» в десктоп-приложении приходит из **облака аккаунта**, кэшируется локально, а локальные файлы стенограмм — **не источник правды** для этого списка. Поэтому локальная «хирургия» файлов не меняла сайдбар в принципе.

## Сопутствующие уловы
- **Виновник «потопа»:** один зациклившийся фоновый таск (каждые 25 мин) наплодил ~91% всего мусора сессий → таск удалён, мусор вычищен.
- **Корень «мои диалоги пропали»:** ночные скрипты ставили флаг `isArchived`, который **синхронизировался в облако** и прятал реальные диалоги везде → скрипты навсегда отключены.
- Приложение — из магазина ОС (MSIX), CLI-сессии — дочерние процессы, поэтому «обновить» = полный перезапуск (закроет работу).

## Решение
1. **Прекратить локальную хирургию сайдбара** — это облачный артефакт.
2. Работать через собственный **каталог-кокпит** всех сессий (архив + авторство + сводки + кнопка «продолжить») как настоящий диспетчерский экран.
3. Никакой авто-архивации сессий — оба архиватора отключены навсегда.
4. Обновление дашборда — через планировщик ОС, не через scheduled-задачу Claude (та плодит новую сессию на каждый запуск).

## Урок
Час ресёрча дороже трёх часов починки не по тому слою. Прежде чем резать локально — проверь, где вообще живёт источник правды.

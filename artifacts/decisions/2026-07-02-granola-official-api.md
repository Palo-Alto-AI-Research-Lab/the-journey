# Decision: Granola-пайплайн строим на официальном API

**Проблема.** Granola пишет все звонки Антона, но транскрипты не попадают во Второй Мозг/CRM. MCP-авторизация слетает (~10 дней), headless-рутина на MCP невозможна. Нужно: (1) полный бэкфилл всех встреч, (2) ночная автоподтяжка без человека, (3) гарантия, что Granola реально пишет звонок.

**Что знали до DR (recall).** MCP был LIVE 22.06 (OAuth, аккаунт bb, 75 встреч/мес), слетел; скилл `/granola-sync` построен, не гонялся; SOP фолоуапов Русланы задокументирован; Granola — облако, запись локальна на машине звонка.

**Что дал Deep Research (2026-07-02; полный отчёт вне волта: `E:\Obsidian\_originals\2026-07-02-dr-granola-extraction-pipeline.md`).**
- У Granola с 02.2026 есть **официальный публичный API**: ключи создаются в десктоп-приложении (Settings → Connectors → API keys), Business/Enterprise, эндпоинты List Notes / Get Note / List Folders, 5 rps, cursor-пагинация, до 25 ключей на воркспейс.
- **MCP = browser-OAuth-only**, для ночного ETL непригоден by design → оставить только для интерактива.
- **Локальный кэш / приватный API** — Granola шифрует кэш и прямо отговаривает (v3→v4→v6 ломались) → только break-glass.
- **Push**: folder auto-post → Zapier (проспективно, не ретроактивно). Не обязателен при нашем объёме — ночной poll достаточен.
- ⚠️ **Запись НЕ автостартует**: Granola пишет только если человек открыл заметку/кликнул нотификацию/Quick Note. «Открытое приложение» ≠ «идёт запись». Продуктовой гарантии «всегда слушает» нет — только операционные контроли.
- CSV-экспорт = сверка/бэкап, не источник.
- Если unattended-запись станет жёстким требованием — миграционный fallback: Fathom/Fireflies/tl;dv (у них webhooks + бот).

**Решение (принято 2026-07-02).**
1. **Прод-рельс = официальный API**: ключ из десктоп-приложения (Granola установлена на хабе) → `secrets\granola.env`.
2. **Бэкфилл**: скрипт `E:\Obsidian\_imports\granola\granola_pull.py` — обход папок/заметок с пагинацией, raw JSON снапшоты, markdown-заметки в волт, SQLite-чекпоинты, дедуп по meeting id.
3. **Ночная рутина** в окно 00:30–04:30 Лиссабон: инкрементальный poll (watermark по updated_at + перечитка свежих заметок), 0-LLM; exponential backoff на 429/5xx.
4. **Скилл `/granola-sync` переписать с MCP на API** (та же логика, другой транспорт).
5. **Правило записи** (в Библию): у каждого звонка ровно ОДНА машина-хозяин записи; Granola в автозапуске на всех машинах звонков; вотчер за 5 мин до звонка пингует «открой заметку Granola на <машине>» — потому что автостарта записи в продукте нет.
6. Zapier-push и Enterprise-admin-API — отложено (⚠️ усложнение; спросить Granola про admin-export при случае).

**Риски.** API-поверхность узкая (3 эндпоинта) — если транскрипт не отдаётся Get Note, добираем CSV-сверкой; формат ответов может меняться — храним raw JSON + парсер отдельным слоем; человеческий фактор записи неустраним продуктом.

**Кто что делает.** Клод: скрипт+рутина+скилл+заметки (после получения ключа). Антон/Руслана: создать API-ключ в приложении (2 мин), держать Granola залогиненной на машинах звонков, открывать заметку на звонке (уже их привычка).

# Decision Memo — стартер-кит для друзей

> Альфа-протокол, Шаг 4 (Synthesis). Recall + 2 раунда DR ([[dr-friends-starter-kit-2026-07-02]]) + препарация конкурентов. `origin: anton` — решение Антона, DR = вход.

## Проблема
Раздавать друзьям (разные роли: data scientist, блогер, автор книги, доктор, инженер, дизайнер, сантехник) рабочий Claude Code так, чтобы они не строили путь Антона с нуля и не получали его личных данных, и чтобы новые фичи доезжали удобно.

## Текущие знания (recall)
- Модель «ведомый-спутник» (skill `/follower-onboard`) даёт ТВОЙ мозг/волт → только для доверенной семьи, НЕ для знакомых.
- Нужна модель B: пустой каркас + универсальные скиллы, друг строит СВОЙ мозг, независим.
- Механизм раздачи: Claude Code plugins + GitHub marketplace.
- Переиспользуемое: leak-scan из follower-генератора.

## Результаты DR (оба раунда) — см. [[dr-friends-starter-kit-2026-07-02]]
Форкать не строить · ролевые паки = контент не код · враг = удержание (~39%/1мес; ручной Obsidian мрёт за неделю) · concierge-онбординг обязателен · moat = курация+сопровождение (сам Антон) · приватная раздача · version-pin + песочница · прятать сложность.

## Варианты
- **V1 (ПРИНЯТ) — форк + курация + concierge (AK-47).**
- V2 — широкий публичный дистрибутив/маркетплейс. Отклонён: груз поддержки, хрупкость, преждевременно (single-user/friends scope).
- V0 — просто zip скиллов. Отклонён: нет «нажал обновить», не решает удержание.

## РЕШЕНИЕ (V1)
1. **Форк `AgriciDaniel/claude-obsidian`** (MIT), запиненный на рабочую версию. Не переписываем движок.
2. **2-3 ролевых шаблона волта** (папки/пресеты/примеры/`USER.md` — только КОНТЕНТ) под реальных первых друзей, не спекулятивно.
3. **1 час concierge с каждым другом**: ставлю плагин вместе, первая быстрая победа, чек-ин через 1-2 недели.
4. **Приватный репо**, не публичный маркетплейс.
5. **Апстрим тестирую в песочнице** до раскатки друзьям; их волт бэкаплю перед апдейтом; авто-апдейт выключен.
Moat = Антон (курация + сопровождение), не код. Проект в разы меньше исходной амбиции «построить дистрибутив».

## Риски и лечение
- Version breakage → pin + sandbox-test + rollback-бэкап.
- Отток (churn) → быстрая победа на 1-й сессии + чек-ин 1-2 нед + одно применение за раз.
- Перегруз → Day-1 минимум, продвинутое opt-in.
- Утечка личных данных → leak-scan + detect-secrets + ручной просмотр; ключи вне репо.
- Fork drift (каждый друг кастомит своё) → общий контракт обновления, бэкап перед апдейтом.

## Что дальше (не начато)
Форк claude-obsidian + первый ролевой шаблон под конкретного друга. Тогда же взять Антону лично `defuddle` (быстрый чистый web→markdown).

# Decision Memo — Hermes мульти-вендорный арбитраж (ОТКЛОНЕНО)

> Alpha Protocol (R+DR). Вопрос: строить ли Hermes Agent ([Nous Research](https://github.com/NousResearch), open-source, MIT) как кросс-вендорный диспетчер разных кодинг-CLI, чтобы тратить ВКЛЮЧЁННЫЕ подписочные баки, а не платный API — ради токен-экономии.

## Проблема
Можно ли через сторонний оркестратор гонять родные CLI каждого вендора по их OAuth, чтобы использовать включённую подписку вместо платного API — и собрать из этого экономию + мульти-платформенную автоматизацию.

## Deep Research — премиса мертва по первоисточникам
Бак-вопрос по вендорам:
- **Anthropic — ❌ самый рискованный.** OAuth официально только для родных приложений; третьим лицам нельзя роутить Free/Pro/Max-кредиты за пользователя. На практике сторонние обёртки упирались в «You're out of extra usage». Anthropic ещё весной 2026 срезал такие обёртки от Pro/Max-квот.
- **OpenAI (Codex) — 🟡 чище всех.** «Sign in with ChatGPT» и headless-режим официально поддержаны, входят в план; дальше платные кредиты.
- **Google (Gemini) — ❌ consumer-доступ к CLI закрыт** (миграция на другой продукт).

Гетеро-DAG даёт ОДНУ узкую реальную ценность — **review-разнообразие**: гетерогенная пара (Claude + Codex) ловит дефект в ~**69.8%** ревью против ~**53.1%** у однородной пары. Это не слепой чейнинг, а разные глаза. Один сильный агент при равном бюджете часто не хуже роя.

Hermes как софт: очень молод, семантика фич меняется раз в недели → бремя поддержки на нас; плюс серьёзный security-долг (читаемый credential-store, обходы approval, prompt-injection через входные каналы).

## Решение
1. **НЕ строить Hermes как экономический backbone арбитража** (высокая уверенность по первоисточникам): премиса мертва — политика вендоров враждебна, а Codex мы и так гоним сами.
2. **Строим Codex + Claude ревью-пару** — единственная связка с ИЗМЕРЕННОЙ пользой (69.8% vs 53.1%). Это «достроить нативно», не зоопарк.
3. **Hermes — только изолированная тест-песочница** под отдельный вопрос (само-обучение + авто-постинг), с нулём реальных кредов и волта, тайм-боксом и критерием «убить» = форсит платно / флаг аккаунта / security-инцидент.

## Пойманная мина
Проверка биллинга по пути показала: наш headless-режим идёт по подписке (OAuth), не по платному ключу. Правило «сначала включённые лимиты, платный API — по согласованию» держится. Ресёрч стоил час и сэкономил месяц строительства мёртвого backbone.

## Паттерн
Пятый подряд внешний DR с одним и тем же паттерном: красивый enterprise-стек, написанный «в вакууме», воспроизводит уже отклонённый вариант. **Бери дешёвое и измеримое, тяжёлый стек выкидывай.**

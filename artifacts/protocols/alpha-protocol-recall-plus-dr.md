# 🅰️ Alpha Protocol — Recall + Deep Research перед решением

**Связывающее правило (must):** Для ЛЮБОЙ новой стратегической функции, продукта, бизнес-модели, токеномики, AI-функционала, GTM, рынка, инвестиционной гипотезы или архитектурного решения — **ЗАПРЕЩЕНО переходить к реализации только на основании recall** (наших уже имеющихся данных). Сначала: **Recall → Gap Analysis → Deep Research → Synthesis → Decision Memo**, и лишь потом реализация. Расширяет [[reglament-pered-lyuboy-aktivnostyu-snachala-recall-vsego-volt-i-rag|правило RECALL перед любой активностью]]: recall — необходимый, но НЕ достаточный шаг для стратегических задач.

> 🧒 Простыми словами (для Антона): строим новый корабль — сначала смотрим свои старые карты (Recall), потом спрашиваем у всех моряков мира, где рифы и штормы (Deep Research), и только потом выбираем маршрут. На одних старых картах можно врезаться в скалу, которую другие давно нашли.

## Три уровня (по масштабу задачи)

**Уровень 0 — Quick.** Мелкое: ответить на вопрос, починить баг, маленькая доработка. → Достаточно обычного recall.

**Уровень 1 — Recall.** Среднее: новый модуль, новая гипотеза, новая фича, новый рынок. Обязательно: (1) recall из памяти, (2) recall из внутренних документов, (3) recall из прошлых исследований, (4) формирование гипотез. **Запрещено сразу принимать решение.**

**Уровень 2 — Recall + DR.** Все стратегические решения (см. связывающее правило). Шаги:
1. **Recall** — что мы уже знаем, что уже исследовали, какие были выводы, какие были ошибки.
2. **Gap Analysis** — чего мы НЕ знаем, какие вопросы остались открытыми.
3. **Deep Research** — лучшие практики, свежие статьи/кейсы, конкуренты, провалы, академические исследования, мнение практиков.
4. **Decision Memo** — формат: проблема · текущие знания · результаты DR · варианты решения · риски · рекомендации.

Только после Decision Memo — реализация.

## Триггер и поток исполнения (для агента/Claude Code)

Короткая команда Антона: **`R+DR`** (= **`RDR`** = **`РДР`** — `+`/пробел и регистр не важны, правило 2026-06-16) или **`alpha protocol`** (он же — `альфа` / `alfa`). Увидев её (или начиная стратегическую задачу Уровня 2), агент обязан:

> **Сокращения «Deep Research» (Антон, 2026-06-14):** `DR` = `ДР` = `Резеч` = `Deep Research` — синонимы. Когда Антон пишет любое из них — он имеет в виду **Deep Research** (шаг 3 / выдать DR-промпт). `R+DR` = полный протокол (Recall + Deep Research); голый `DR`/`ДР` = именно шаг Deep Research. Контекст решает (если неоднозначно — переспросить; «ДР» бывает и «день рождения»). Для людей-ассистентов то же: «сделай ДР по X» = «проведи Deep Research по X».
>
> **⭐ ВАЖНО — кто исполняет DR (Антон, 2026-06-22, origin: anton):** «сделай ДР / DR / Deep Research» **ВСЕГДА = я (агент) выдаю ПРОМПТ для ВНЕШНИХ ЛЛМ, включая Cowork**. Я НЕ провожу ресёрч сам. Дефолт = хэндофф наружу. **Я делаю ресёрч сам («инхаус») ТОЛЬКО если Антон явно пометил** одним из: **`ИДР`** / **`INHOUSE DEEP RESEARCH`** / **`LIGHT RESEARCH`** / **`LR`**. Без этой пометки бежать в WebSearch/полный ресёрч самому = ОШИБКА (грабли 2026-06-22: на «давай сделаем ДР» я ушёл делать сам вместо выдачи промпта). Узкий «лёгкий веб-проход, чтобы заточить ВЫДАВАЕМЫЙ промпт» остаётся разрешён — но это не замена хэндоффа.

1. **RECALL** — поднять всё своё (память + grep по волту + RAG `/ask` + SQLite-факты).
2. **GAP** — сформулировать, чего не хватает / открытые вопросы.
3. **Выдать DEEP RESEARCH PROMPT** — заполненный [[deep-research-prompt-template|шаблон]] (подставить `{topic}`, при наличии `CONTEXT:` из recall). **DUAL-TRACK — два параллельных трека Deep Research (Антон, 2026-06-21; УТОЧНЕНО 2026-06-21 — БЕЗ робот-вождения):** агент даёт Антону ДВА промпта, **Антон сам их вставляет** — (а) в его ВНЕШНИЙ инструмент (Gemini / ChatGPT Deep Research), И (б) в **Cowork** (отдельный больший бак — бережём бак Claude Code). ⛔ Агент **НЕ** диспатчит Cowork-задачи на десктоп-хаб для автономного GUI-вождения: бак Cowork тратится одинаково, кто бы ни нажал «вставить», а ручная вставка проще и надёжнее хрупкого робота (АК-47). *(Гибрид: агент может сделать лёгкий веб-проход своими инструментами, чтобы заточить промпт.)* Итог: глубокий DR делается В Cowork, но запускает его Антон вставкой — НЕ агент автономным вождением GUI.
4. **SYNTHESIS** (когда Антон принёс результат внешнего DR + результат Cowork, который он сам вставил) — объединить память + ОБА независимых результата DR → Decision Memo → рекомендация → решение. Два независимых прохода = больше уверенности, меньше слепых зон.

# Executive Summary  
Claude Code's plugin/skill ecosystem has multiple discovery channels, each with distinct reach and mechanics. The official Anthropic "claude-plugins-official" marketplace offers the highest trust and adoption (with top plugins in the **100k+ install** range) but is heavily curated and gated. The large community-driven marketplaces (Alireza Rezvani's **claude-skills** and Jeremy Longshore's **Tons of Skills**) have significant usage – roughly **184k total installs** for alirezarezvani's 266 skills and **14.1k monthly downloads** across 453 plugins for Jeremia's site – and enable direct CLI installation. The CrossAI Tools (formerly *claudemarketplaces.com*) directory receives **~300k monthly visits** and automatically indexes thousands of skills and marketplaces, offering broad exposure. The "awesome" list by travisvn is a high-profile GitHub resource (13.9k stars) but is not a distribution medium. In practice, new third-party tools should aim first for **Anthropic's official marketplace (if eligible), then community-run marketplaces (alirezarezvani, Longshore), and ensure inclusion in the CrossAI directory**, supplemented by the Awesome list for visibility among devs. Each channel has its own submission requirements (e.g. Anthropic's security checks, Jeremia's 8-field frontmatter and test suite, Travis's "social proof" rule). **All listings must be free**; none of these channels support paid placements. Support and churn vary: Anthropic maintains the official channel, community tools rely on their maintainers (though Longshore and Travis have active contributors), and CrossAI Tools is auto-updated.  

# Key Findings  

- **Official Anthropic Marketplace** (anthropics/claude-plugins-official): This is a **high-trust, curated repository** of plugins managed by Anthropic. It has **tens of thousands of stars (31.4k) and forks (3.4k)**, and **individual plugins** show **hundreds of thousands of installs** on Claude (e.g. GitHub MCP: 291,655; Slack MCP: 80,897). Discovery is built-in: users can browse the Anthropic plugins UI or run `/plugin discover`. **Listing mechanics:** Third-party tools must apply via Anthropic's submission form; all submissions undergo automated screening and manual review, with successful ones earning an "Anthropic Verified" badge. Adoption data is private to Anthropic, but the public UI reveals total install counts. No monetization – the directory is free. Ongoing churn/support is managed by Anthropic; developers have no obligation after listing, and usage is tracked internally.  

- **Alireza Rezvani's Claude Skills** (alirezarezvani/claude-skills): A **large community marketplace** of Claude "skills" (266 skills listed). It boasts **~19.6k GitHub stars** and, according to CrossAI Tools, **184.3k total installs** across those skills. Users discover it by adding the marketplace via `/plugin marketplace add alirezarezvani/claude-skills` and installing skill bundles (e.g. `engineering-skills@claude-code-skills`). **Growth:** It appears to have grown organically via GitHub visibility and community sharing. Submission is by PR to Alireza's repo (he encourages contributions). There is no formal vetting beyond Alireza's own review and automated checks (many listed skills have automated tests). It's entirely free. Support is community-driven; Alireza maintains releases (latest v2.9.0 in May 2026). Downside: as a personal collection, it may not filter out low-value or duplicate content as strictly as an official channel.  

- **Longshore's TonsOfSkills (jeremylongshore/claude-code-plugins-plus-skills)**: An open-source **"marketplace" with CLI**. It contains 453 plugins/3,032 skills and publishes them via NPM packages (425 packages). Usage stats: 14,130 monthly NPM downloads (all packages), and CrossAI Tools sees **7.5k total installs across 13 listed skills** (note: CrossAI only shows a subset). Discovery: users can add the marketplace with `/plugin marketplace add jeremylongshore/claude-code-plugins` or install via the `ccpi` CLI. Growth has been both **seeded and organic** – Longshore's own high-quality contributions and a community (30 contributors) contribute content. A strict 100-point rubric and automated validators ensure quality: every PR is tested against an 8-field frontmatter schema (name, description, tools, version, author, license, compatibility, tags). No paid listings. Maintenance is active (daily updates via CI), but unsupported plugins may stagnate.  

- **CrossAI Tools (claudemarketplaces.com)**: An **automatically updated directory** of Claude marketplaces, skills, and MCP servers, run by a third party (Mert Düzgün). It reports **300k+ monthly visitors** and indexes ~21,600 skills and 2,500 marketplaces. It **aggregates** all public marketplaces/skills (by scanning GitHub for valid `marketplace.json` or `SKILL.md`). It displays install counts and star rankings (e.g. trending skills with millions of installs) and curates listings by usage: "only actively used extensions are listed". It's free (no listing fee) and supports ad revenue (300k visitors, sponsored placements). Discovery is via website search or browsing. Developers cannot "submit" directly; their repos simply appear if they have a proper schema. No vetting beyond usage metrics: low-usage items may be omitted. Churn is minimal since it's auto-synced; no maintenance needed from listing owners.  

- **Travisvn's Awesome Claudes Skills**: A **curated GitHub list** (13.9k stars) of noteworthy skills and resources. It's not a distribution platform – it links to external repos. Discovery is via GitHub search or community references. Growth is purely organic (community PRs, 532 open PRs shows active contributions). **Vetting:** Travis imposes strict submission rules: skills must provide "clear value" and **social proof** (at least ~10 stars) or PRs are closed automatically, and "promotional" or SaaS-specific submissions are rejected. There is no monetization. Support is via community PRs; it's a living doc that only changes when contributors update it. A tool listed here gains visibility to developers but no direct install mechanism.  

# Contradictions & Debates  

- **Quality vs. Quantity:** Community lists (CrossAI Tools, Awesome-claude) emphasize quantity and usage metrics, whereas curated marketplaces (Anthropic, TonsOfSkills, Travis) emphasize quality. For example, Travis's "no ads or SaaS wrappers" rule ensures high signal but excludes commercial/sketchy entries; in contrast, CrossAI Tools will list any plugin/skill that meets the format and has usage. This raises debate: a highly differentiated tool with low initial usage may be excluded by Travis's social-proof rule or buried in CrossAI's data, while generic clones might sneak through channels without strict vetting.  

- **Stars vs. Installs as Popularity:** Travis's "social proof" requirement suggests GitHub stars indicate usefulness, but usage tells a different story. E.g., Alireza's marketplace has ~19.6k stars and 184k installs, whereas the Awesome list has 13.9k stars but no install metric. Conversely, CrossAI Tools highlights some million-install skills that may have few stars. The community debates whether star counts or download analytics better measure reach; likely both should be considered.  

- **Open vs. Closed Ecosystem:** Anthropic's official marketplace is gated (screened by automated and manual review) to maintain trust. Some argue this ensures reliability, while others note it limits exposure: Reddit users found 50+ hidden official plugins by inspecting configs. In contrast, open channels (Alireza, Longshore) may seed content faster. The tension is between "curated trust" (official/verified badges) and "open discovery".  

- **Monetization Models:** All these channels are free for developers, but monetization exists in hidden forms. CrossAI Tools carries ads/promotions to sustain operations. This raises debate: paid ads on the directory might boost revenue but could compromise neutrality. Travis explicitly forbids promotional submissions to keep it free of marketing. There's a balance: free listing maximizes adoption, but resources for curation must come from somewhere.  

# Best Practices  

- **Prepare High-Quality Plugin Packages:** For any marketplace, ensure your plugin or skill is fully production-ready: include a **plugin.json or SKILL.md**, comprehensive README, examples, and tests. Follow schema conventions: e.g. TonsOfSkills enforces an 8-field frontmatter (`name`, `description`, `allowed-tools`, `version`, `author`, `license`, `compatibility`, `tags`). Use the official IntentSolutions validator or Claude's CLI `ccpi validate` to catch issues. Provide a clear description, licensing, and compatibility range to meet each channel's expectations.  

- **Meet Channel-Specific Entry Criteria:** Tailor submissions to each listing's rules:
  - **Anthropic Official:** Adhere to Anthropic's plugin security and quality guidelines. Use the submission form on claude.ai (clau.de) and ensure automated checks pass. Provide audit logs or privacy reviews if applicable; high-security content (e.g. banking tools) may need extra vetting.  
  - **Alireza's Marketplace:** Follow his repo's contribution guide. Submit as a new skill or plugin with complete documentation. Even if not required, including tests and following common best practices will improve acceptance.  
  - **Longshore's Marketplace:** Follow the 100-point rubric (publicly documented). This means providing complete metadata, valid schema, and passing all CI checks. Use the `ccpi` CLI to test installation (e.g. `ccpi install <your-plugin>`) and ensure it works. Submit via PR to his repo with the `marketplace.json` entry or add your plugin in the appropriate folder.  
  - **CrossAITools:** Ensure your project includes a valid `marketplace.json` or Claude Schema so the directory can auto-detect it. If listing a single skill, include a `SKILL.md` at the repo root. Proper use of the official schemas is the only requirement; there is no application process.  
  - **Travis' Awesome List:** Submit via PR on GitHub. Ensure your tool is genuinely useful (not just a branded wrapper). Provide a markdown entry (e.g. `- **[Name](link)** - Description`). Crucially, make sure the project has at least **~10 GitHub stars** or evidence of community interest, or your PR will likely be closed automatically. Avoid any promotional tone – the entry should be purely educational/resourceful.  

- **Leverage Multiple Channels:** Don't rely on one listing. Cross-posting on all relevant channels maximizes exposure. For example, push a PR to the Awesome list for visibility among devs, add your marketplace for CLI installs, and let CrossAI aggregate it. Keep all listings synchronized (e.g. plugin.json versions should match).  

- **Maintain and Support:** Be prepared for upkeep. Official and community marketplaces may require updates as Claude evolves. Provide issue support if users encounter problems. For official listings, Anthropic may update the base plugin directory, so periodically re-validate compatibility. For community repos, respond to PR comments and keep your submission up-to-date to avoid becoming stale.  

# Failure Modes  

- **No Traction on Listings:** Tools with no users often fail to gain visibility. On CrossAI Tools, extensions with near-zero installs simply don't rank (its algorithm "only actively used extensions are listed"). On the Awesome list, submissions without stars are auto-rejected. The cure is to build an initial user base or demo case first, then submit. Failing to do this means the tool remains invisible.  

- **Ignoring Submission Rules:** Submissions that violate channel rules get ignored or removed. For example, Travisvn rejects "SaaS wrapper" tools and overt marketing pitches. If you try to list a paid-only integration or a promotional skill, you'll waste effort. Similarly, Jeremia's validator will reject any plugin failing format (e.g. missing frontmatter). Thoroughly read and follow each channel's guidelines before submitting.  

- **Poor Quality or Documentation:** If a plugin's README is sparse or the code is flaky, user trust evaporates. On official channels, a bug or security issue could block listing. On community channels, poor documentation means nobody installs it. Claude plugins especially must pass security checks, so leaving open credentials or ambiguous behavior can kill adoption.  

- **Clones and Generic Tools:** The landscape is crowded with "PKM clone" skills (personal knowledge managers, etc.) that offer little new value. Channel curators filter these heavily. The Awesome list explicitly requires "clear value" and non-trivial functionality. If your tool isn't genuinely differentiated (e.g. just another generic note-taker skill), it will drown in the noise and likely be ignored by both users and curators.  

- **Lack of Long-Term Support:** Community marketplaces rely on maintainers. If Longshore or Alireza stop updating, listed plugins may break on new Claude versions. Abandoned tools lose compatibility. Anthropic's official plugins also get updates; if a third-party plugin is not maintained, Anthropic may delist it. Plan for at least minimal maintenance (e.g. fix build failures on new releases).  

# Strategic Recommendations  

1. **Anthropic's Official Marketplace:** *Priority #1.* It has the largest built-in user base and the highest trust. Although acceptance is stringent, an approved listing here guarantees exposure to all Claude Code users and access to Anthropic's verified badges. Target this channel if your tool meets Anthropic's security/quality criteria (especially for enterprise or high-sensitivity tools). Provide complete `plugin.json`, thorough documentation, and any required audits. The downside is a longer review cycle, but the payoff is unmatched reach (e.g. GitHub plugin = 291k installs).  

2. **Alireza Rezvani's Claude Skills:** *Priority #2.* With 184k installs for 266 skills, this marketplace has proven popularity. Its CLI install mechanism is as easy as the official one. Contribute a skill via PR (following his repo style) or bundle your functionality as a skill. This taps into a large user community that follows Alireza's work. Because it's free and open, listing is relatively straightforward. Ensure your skill fits his structure and documentation quality (see his existing skills).  

3. **TonsOfSkills by Jeremy Longshore:** *Priority #3.* Though smaller (7.5k CrossAI-tracked installs), this channel offers advanced tooling and engaged audiences (with 14k monthly npm downloads). Use the `ccpi` CLI to validate and submit your plugin. Follow the exact schema and adhere to the 100-point rubric (see his repo). This channel filters out low-effort entries, so only submit if you can meet its high standards – but if you do, the quality-conscious user base will notice.  

4. **CrossAI Tools Directory:** *Priority #4.* This aggregator has **300k+ monthly visitors**. It's an easy win: any properly formatted marketplace or skill repository will be indexed automatically. To leverage it, ensure your GitHub repo has the correct Claude marketplace/skill schema. There is no official submission, but you can flag it to the maintainer if needed. Because it curates by install count, mention your install instructions in documentation so users know how to install (which will bump its usage metrics).  

5. **Awesome-Claude-Skills List:** *Priority #5.* Use this for visibility among developers. Submit your tool (if it's a "skill" or tutorial) via PR and ensure it meets Travis's rules (non-promotional, useful, with >10 stars). This won't directly drive installs, but it signals quality to the community. It's a good ancillary listing after the above channels; even if your tool is unique, requiring social proof means you should build some credibility (e.g. community engagement, docs) before requesting inclusion.  

# Action Plan  

1. **Develop to Channel Standards:** Before listing, meet each channel's baseline. For Anthropic, create a `plugin.json` with name/description, specify tools (SKILL or API), set semantic version, author, license, compatibility (Claude Code v2.x), and topics. Include a `SKILL.md` or examples demonstrating functionality. For TonsOfSkills, write an 8-field frontmatter JSON and tests so that `ccpi validate` passes. For CrossAI, ensure your repo has a valid `marketplace.json` (with `plugins` or `skills` entries) or `SKILL.md` in root.  

2. **Prepare Documentation:** Write clear README and usage instructions. Cite evidence (meta-analyses, RCTs) if applicable, since the user is evidence-driven. List dependencies and limitations. If possible, include a minimal code snippet showing CLI install commands (e.g. `/plugin install your-tool@claude-plugins-official`).  

3. **Submit to Official Marketplace:** Use the official submission form at `claude.ai/plugins` (or anthopic site) to propose your plugin. Provide all required info (names, links to source code, a security overview). Respond quickly to any follow-up from Anthropic.  

4. **Add to Community Marketplaces:** Fork each relevant GitHub repo and submit a PR:
   - For alirezarezvani/claude-skills: add your skill in the appropriate category folder (or follow contributing docs).  
   - For jeremylongshore: add your plugin in `marketplace.json` or `plugins/` folder as specified, with validated metadata.  
   - For Travisvn's awesome list: add a markdown entry under the right section, following format. Mention any GitHub stars or usage you have as social proof.  

5. **Check Aggregator Indexing:** After your repos are published, visit CrossAI Tools (Search or "Marketplaces" tab) to verify listing. If missing after a day, consider filing an issue on their repo or contacting the maintainer to ensure they detected your marketplace JSON.  

6. **Promote and Maintain:** Share your tool on Claude community channels (Discord, Reddit, etc.) so "real strangers" see it. Encourage users to star or download (to boost algorithmic ranking on Travis/CrossAI). Monitor feedback on each channel and update your listing if needed (bug fixes, new features). Consider setting up analytics (like a GitHub release note or website hits) to gauge impact.

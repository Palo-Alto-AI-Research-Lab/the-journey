# Инсайт — чат это не демон

> Из честного вопроса оператора: «почему тебя нужно пинговать, ты не видишь звонок сам?» ([День 24](../../02-june-scaling/week-4/2026-06-23.ru.md)).

## Суть
Claude Code — процесс **по требованию**, не фоновый демон. Он видит событие (звонок в календаре, файл приехал) прекрасно — но как чат-сессия «просыпается» только когда ему пишут. Между сообщениями его нет.

## Следствие для архитектуры
Для круглосуточной автономной слежки нужен **отдельный вечноживой процесс** — и жить он должен на **всегда-включённой машине** (хабе), а не на ноутбуке, который уходит спать вместе с человеком. Это разводит две вещи, которые легко перепутать:
- «Агент способен прочитать календарь по запросу» — да, в чате.
- «Агент следит 24/7 и сам толкает работу» — нет, для этого нужен демон/диспетчер.

## Как решаем
- Тяжёлое и постоянное → на хаб («минимум ноута, максимум десктопа»).
- Единый диспетчер сигналов (не чат) будит нужное по событию; дефолт — пинг человеку.
- Ноутбук — для живого интерактива, не для фоновой службы.

## Почему это честно
Агент, который говорит «я вижу событие, но просыпаюсь только когда ты пишешь», надёжнее агента, который делает вид, что дежурит. Знать границу собственной автономии — часть автономии.

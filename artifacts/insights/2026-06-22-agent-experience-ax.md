# Концепт — Agent Experience (AX)

> Новый узел графа, вызревший из главной истории [Дня 23](../../02-june-scaling/week-4/2026-06-22.ru.md).

## Что это
**AX (Agent Experience)** — термин, введённый CEO [Netlify](https://www.netlify.com/) Матиасом Билманом в 2025: как продукт выглядит и работает для AI-агента, а не для человека. Если UX — про кнопки и экраны, то AX — про протоколы, схемы и права доступа, которые агент понимает без интерфейса.

## Почему это важно
Мир переходит от «человек нажимает кнопки» к «агент разговаривает с агентом». Клиент будущего может не иметь рук — у него есть агент, который сам ходит по чужим системам. Тогда:
- **Интерфейс не нужен — нужен протокол** (MCP, A2A, AP2, x402).
- **Продукт — не CRM, а привратник:** чужой агент приводит задачу, твой агент решает, открывать ли дверь, и исполняет на твоей стороне под апрувом.
- **Ров — не технология, а доверие людей.** Софт перепишут за неделю; поверхность распространения (тёплые контакты + право на тёплое интро) — нет. Именно её не закрывают гиганты.

## Как применяем
Строить не «ещё одну CRM с ИИ», а **брокер-слой** поверх собственной сети доверия: read + request наружу, execute — только свой аккаунт человека. Держать ключ от двери у себя.

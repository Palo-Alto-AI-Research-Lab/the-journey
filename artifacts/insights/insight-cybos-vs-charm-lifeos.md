# CYBOS ↔ CHARM / LifeOS — сверка и что забрать

> [!abstract] Зачем
> Антон: «CYBOS ОЧЕНЬ ПОХОЖ на CHARM — взять как можно больше его мыслей». Это карта совпадений двух «персональных ОС / цифровых двойников» и список конкретных паттернов Гершуни, которые можно перенять в [[concept-charm-lifeos|CHARM / LifeOS]]. Источник: 2 Deep-Research-отчёта + [[insight-ai-native-playbook-gershuni|AI-native плейбук]] + частотный анализ 731 эссе.

## Две системы рядом
| Ось | **CYBOS** (Степан Гершуни) | **CHARM / LifeOS** (Антон) |
|---|---|---|
| Суть | Personal OS для жизни и работы | Цифровой двойник из данных человека (Anima) + relationship-инфра |
| Ядро | Единый контекст + GTD + оркестрация ресёрча + агенты, имеющие доступ к функциям | Память-граф ([[Anima]]) + поведенческая модель + AI-last-mile поверх переписки |
| Главный механизм | **Recursive-correction-loop**: ежедневные правки впитываются обратно в инструкции | CRM/агенты на личных данных; двойник звучит и действует как человек |
| Вектор | AI-native компания/человек; cybereconomy | **Digital immortality** + biological longevity ([[Anima]]+[[Vital]]) |
| Вход | голос/текст | голос-first ([[concept-graphomania-voice-first-writing]]) → текст |
| Дистрибуция | community «монастырь» (cybos.ai) | мессенджеры (TG/WA/Signal/Slack/Discord) |

**Вывод:** это два независимых воплощения одной идеи — *человек → единая контекстная машина → автономный агент-двойник*. CYBOS сильнее в **operational-слое** (как жить внутри системы каждый день), CHARM — в **векторе** (бессмертие/наследие) и **relationship-инфре**. Они дополняют друг друга.

## ⭐ Что забрать из CYBOS в CHARM (приоритет)
1. **Recursive-correction-loop как явный механизм продукта.** Не просто «память», а контур: правка пользователя → переписывание инструкции/skill → класс ошибок не повторяется. Это конкретная фича для [[Anima]].
2. **«Жить в одной контекстной машине, а не ходить в новый чат».** UX-принцип №1 для CHARM: единый контекст, а не сессии.
3. **Process-as-product (evals + версии + owner + data flywheel).** Превратить внутренние процессы CHARM (аутрич, ресёрч, контент) в версионируемые артефакты с метриками.
4. **6 стадий AI-native** как модель зрелости — продать клиенту CHARM не «тулзу», а путь cost-opt → scaling → cybernetic self-improvement.
5. **Агент с экономической субъектностью** (платежи/мандаты/supervised autonomy — «openclaw купил машину») — дорожная карта для «двойник, который действует, а не только советует» ([[insight-ai-klon-agent-deystviya-ne-chat-bot]]).
6. **Стадии и «no-slides» онбординг** (cybos.ai показывает реальные workflow, а не презентации) — формат для community/демо CHARM.
